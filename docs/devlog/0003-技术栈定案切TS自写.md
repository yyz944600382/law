# 0003 · 技术栈定案:从 deerflow 切到 TS 纯自写

- **日期**:2026-06-25
- **类型**:技术决策(架构方向)

## 背景
spec-001 原定用 deerflow 开发(见 devlog/0002)。推进中发现 deerflow 无 PyPI 包、必须 clone 整个应用才能用其 SDK,过重。

## 评估过程
1. deerflow:否决(上述)。
2. 询问 python 多 agent 框架推荐(CrewAI/LangGraph/AutoGen)→ 用户倾向 TS,指引看本地 clowder-ai。
3. Explore clowder-ai:是**产品级重平台**(Redis/SQLite/Fastify/Next.js/Electron + 外部 CLI 适配),深度耦合,**不可直接复用**。但用纯 TS 验证了 4 件关键事:
   - `LlmAIProvider.ts`:程序内直连 OpenAI 兼容 API(改 baseURL 接 DeepSeek)
   - `CatAgentService.ts`:agentic tool loop + SSE
   - `WerewolfEngine.ts`:**LLM 出意图 + 纯代码裁决**(对法官极有启发)
   - `a2a-mentions.ts`:消息路由 + 轮次调度
4. 用户拍板:**TS 纯自写**(参考 clowder-ai 模式,不引框架、不搬其代码)。

## 决策
- TS 纯自写,npm,DeepSeek,免 key 联网检索,deterministic JudgeEngine,CLI demo,vitest 禁 mock。
- 关键创新:法官 = 确定性 JudgeEngine + LLM 结构化输出(控幻觉底线)。

## 影响 / 后续
- 所有文档从 deerflow/python/uv/Docker 切到 TS(本次 `docs/pivot-to-ts`)。
- 计划书归档 `docs/plans/0001-ts-rewrite-plan.md`。
- 实现按该计划的 8 步推进。
