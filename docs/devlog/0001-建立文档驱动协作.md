# 0001 · 建立文档驱动的协作体系

- **日期**:2026-06-25
- **类型**:流程 / 约定

## 背景
项目启动初期,确立协作方式:用户写简略 spec,Claude 理解后实现;迭代过程与解决问题的经验都沉淀成文档,避免重复沟通与踩坑。

## 决策
1. 建立 `docs/` 三件套:
   - `docs/specs/` —— 需求规格(用户写,Claude 实现)
   - `docs/devlog/` —— 开发日志(Claude 维护)
   - `docs/lessons/` —— 经验沉淀(Claude 写入)
2. **CLAUDE.md 随对话持续更新**:对话中确定的新约定、新事实及时写入。
3. 协作循环:用户写 spec → Claude 实现 → 追加 devlog → 必要时沉淀 lesson。

## 影响 / 后续
- 后续所有需求经 `docs/specs/` 流转;所有经验经 `docs/lessons/` 沉淀。
- 遇到问题先查 `docs/lessons/` 与已有 spec,再动手。
