# Specs · 需求规格

存放需求规格文档。**你写简略 spec,Claude 据此理解需求并实现。**

## 如何提交一个 spec

1. 新建文件:`NNNN-简短描述.md`,编号从 `0001` 递增。
2. 用下面的模板填写,不必详尽 —— 写清「目标」和「验收标准」即可,Claude 会主动澄清模糊处。
3. 在文件顶部标注状态:`草案` / `已确认` / `实现中` / `已完成`。

## Spec 模板

````markdown
# <spec 标题>

> 状态:草案

## 目标
<一句话:这个需求要达成什么>

## 背景 / 动机
<为什么要做?可选>

## 范围
- 做:<...>
- 不做:<...>

## 验收标准
- [ ] <可验证的完成条件>

## 备注 / 约束
<可选:性能要求、参考资料、特殊限制>
````

## 与其他文档的关系
- 实现过程记录在 `../devlog/`。
- 实现中遇到的问题沉淀到 `../lessons/`。
