# Lessons · 经验沉淀

结构化的问题解决记录。**遇到任何问题,先在这里检索**,避免重复踩坑。

## 如何写一条 lesson

新建文件:`NNNN-简短问题描述.md`,编号递增。

## 建议结构
- **问题现象**:出了什么问题 / 报什么错
- **根因**:为什么会这样
- **解决方案**:怎么解决的(关键命令 / 代码 / 配置)
- **如何复现 / 触发条件**(可选)
- **相关链接**:关联的 spec、devlog、外部资料

> 与 `../devlog/` 的区别:lessons 是按「问题」组织的知识卡片,devlog 是按「时间」组织的叙事日志。
