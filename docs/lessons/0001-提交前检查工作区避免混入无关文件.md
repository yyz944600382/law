# 0001 · 提交前检查工作区,避免混入无关文件

## 问题现象
执行 `git add -A` 后提交,把工作区里**用户自行新建的空文件** `docs/specs/spec-001-simple-demo.md` 一起带进了 commit,导致该 commit 混入了与本任务无关的文件,违反「一个逻辑变更一个 commit」。

## 根因
`git add -A` 会暂存工作区**所有**改动,包括 Claude 未主动关注、由用户自行创建的文件。提交前没有先 `git status` 检查,无法察觉意外文件。

## 解决方案
- **提交前先 `git status --short`** 查看将纳入的改动。
- **按需精确 add 指定文件**(`git add <具体文件>`),而非无脑 `git add -A`;仅当确认工作区只有目标改动时才用 `add -A`。
- 发现混入时:尚未 push 可 `git reset HEAD~` 拆开重提;已 push 则记录、后续注意。
